Crypt Tool Commands

The crypt tool provides two primary commands:

1. keygen Command

Purpose: Generate a key file of a specified size, either randomly or deterministically using a password.

Usage:

crypt keygen <size> --mode <random|deterministic> [--password <password>]

Arguments:

  • <size>: The size of the key file to generate. Examples:
    • 32bytes
    • 20mb
    • 5gb
  • --mode: The mode of key generation.
    • random: Generates a key with random data.
    • deterministic: Generates a key based on a password.
  • --password (optional): Required if --mode is set to deterministic. This is the password used to generate the deterministic key.

Examples:

# Generate a 32-byte random key
crypt keygen 32bytes --mode random

# Generate a 20 MB deterministic key with a password
crypt keygen 20mb --mode deterministic --password mypassword

2. xor Command

Purpose: Perform XOR encryption or decryption on a file using a key file.

Usage:

crypt xor <input_file> <output_file> <key_file>

Arguments:

  • <input_file>: The path to the input file you want to encrypt or decrypt.
  • <output_file>: The path where the output (encrypted or decrypted) file will be saved.
  • <key_file>: The path to the key file that will be used for the XOR operation.

Examples:

# Encrypt a file
crypt xor mydocument.txt encrypted_output.bin key.key

# Decrypt a file
crypt xor encrypted_output.bin decrypted_document.txt key.key

Notes

  • The keygen command generates a key file named key.key in the current directory. If a file named key.key already exists, the command will display an error.
  • When using the xor command, ensure that the key file is at least as large as the input file.
